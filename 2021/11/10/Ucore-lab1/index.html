为何开启A20以及如何开启A20？

为何开启A20？

在8086CPU中，16位寄存器，20位地址线，通过内存分段可以访问1MB的内存，但是内存分段可以访问的内存最大值是0xffff:0xffff，超过了1MB，但是因为只有20个地址引脚 $A_0-A_{19}$，超出的高端内存区就会自动舍弃，称为地址回卷。

在80386CPU中，32位寄存器，32位地址线，保护模式下可访问4GB内存，实模式下可以访问1MB。进入到保护模式之后需要访问更大的内存，如果还存在地址回卷的话是无法访问的，所以需要关闭地址回卷。A20是用来控制回卷特性的，为了向下兼容8086，需要可以存在回卷和不回卷两种状态。

如何开启A20？

打开A20地址线控制，需要向键盘控制器8042发送一个命令来完成，8042会把某个输出引脚的输出置高电平作为A20地址线控制的输入，就开启了A20地址线。

8042有4个寄存器：

  • 1个8-bit长的Input buffer；Write-Only；
  • 1个8-bit长的Output buffer； Read-Only；
  • 1个8-bit长的Status Register；Read-Only；
  • 1个8-bit长的Control Register；Read/Write。

8042有两个端口60h,64h，有4个操作：

  • 读60h端口，读output buffer
  • 写60h端口，写input buffer
  • 读64h端口，读status register
  • 操作control register，先向64h写一个命令：20h为读命令，60h为写命令，然后根据命令从60h端口读出Control register的数据或者向60h端口写入Control Register的数据。

此外8042还有三个内部端口Input Port, Output Port, Test Port，操作这三个端口都是向64h发送命令，在60h完成读写，A20 gate就在Output的bit 1上。

开启A20date的步骤：

  1. 等待input buffer为空
  2. 发送写output port的命令到64h端口
  3. 等待input buffer为空
  4. 把ouput port bit设置为1，发到60h端口

[练习4]分析bootloader加载ELF格式的OS的过程

  1. bootloader如何读取硬盘扇区的？

    bootmain()函数调用readseg()函数来读取磁盘中的分区到指定内存处

    readseg()调用readsect()函数来依次读取扇区。

    磁盘数据是存储到硬盘扇区中的，一个扇区512字节。读一个扇区的流程的流程大致如下：

    1. 等待磁盘准备好
    2. 发出读取磁盘的命令
    3. 等待磁盘准备好
    4. 读取磁盘数据

  2. bootloader如何加载ELF格式的OS？

    readseg()函数把磁盘数据（kernel）读取到ELFHDR，也就是0x10000处。

[练习6]完善中断初始化和处理

  1. 中断描述符表（也可简称为保护模式下的中断向量表）中一个表项占多少字节？其中哪几位代表中断处理代码的入口？

    8个字节。其中16-31位是中断处理代码段描述符的选择子，0-15和46-63位是中断处理代码在目标段内的偏移。

    需要根据选择子去GDT表中索引，找到对应段的起始地址，加上段内偏移才是中断处理代码的入口地址。

  2. 请编程完善kern/trap/trap.c中对中断向量表进行初始化的函数idt_init

    其中SETGATE是一个宏，其中__vector[]数组存储的就是中断处理程序的偏移，设置对应的DPL和段选择子。

    使用lidt来设置IDTR寄存器，这个函数是内嵌汇编实现的。

  3. 请编程完善trap.c中的中断处理函数trap.

    直接定义一个全局变量记录时钟中断的次数，到100次的时候清零并调用print_ticks()函数就可以。
